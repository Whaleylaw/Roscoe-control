# Review: Final Distribution Statement

Approve only if the worker prepared a reconciled statement without crossing into actual distribution.

Review the proposed diff/worktree changes, not merely historical files already
present on the base branch. Existing downstream final-distribution or close-case
logs from a prior branch test are not a rejection reason unless this task added,
edited, or relied on them as current completion evidence.

The worker must:

- use canonical settlement/distribution/lien evidence and same-task comments for missing test facts,
- produce a final distribution statement draft,
- update settlement/distribution ledgers as prepared for human issuance only,
- preserve no-lien or lien-payment boundaries,
- append new activity and workflow-log entries without modifying prior log entries.

Reject if the worker invents financial facts, ignores reconciliation errors,
edits existing activity/workflow-log entries, marks the client paid, zeroes the
trust account, closes the case, marks final distribution complete, or adds new
downstream completion/closure artifacts in this task.
