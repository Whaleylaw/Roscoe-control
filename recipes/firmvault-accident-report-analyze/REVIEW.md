# Review: Accident Report Workflow

Approve only if the worker completed the assigned accident-report node narrowly and used canonical FirmVault paths.

For status nodes, approve only if the worker checked canonical report paths first, avoided duplicate requests when a report was already present, and documented the status with evidence.

For request nodes, approve completion only if the worker either found the canonical report already present, documented owner-confirmed request evidence, or prepared a specific request handoff with enough agency/request facts for a human to act. If the worker only discovered that agency/report/request details are missing, the correct verdict is blocked, not approved.

For analysis nodes, approve only if extracted facts are traceable to the report shadow and written to contract-approved accident, contact, and insurance ledgers.

Reject if the worker overstates liability, invents insurance details, invents report agency/number facts, misses obvious party/provider information from the report, or writes outside the case worktree.
