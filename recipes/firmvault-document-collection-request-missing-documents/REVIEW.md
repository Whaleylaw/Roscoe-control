# Review: Missing Onboarding Document Request

Approve only if the worker prepared a clear, auditable handoff for missing onboarding documents and kept the work inside the mounted FirmVault case.

Reject if the worker assumed signatures, wrote raw PHI, contacted anyone externally, changed unrelated case files, or skipped canonical status files before drafting the request.

The review should verify:

- The worker checked whether the missing documents were already present.
- The missing items are stated precisely.
- The handoff is useful for a human sender without exposing raw PHI.
- Activity and workflow-log entries explain what was checked and what changed.
- The diff is limited to the relevant case folder and this workflow step.
