Migration 061 — task_runner_attempts:
```typescript
{
  id: '061_task_runner_attempts',
  up(db: Database.Database) {
    db.exec(`
      CREATE TABLE IF NOT EXISTS task_runner_attempts (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        task_id INTEGER NOT NULL,
        attempt INTEGER NOT NULL,
        started_at INTEGER NOT NULL,
        exited_at INTEGER,
        exit_code INTEGER,
        failure_reason TEXT,
        stderr_tail TEXT,
        created_at INTEGER NOT NULL DEFAULT (unixepoch()),
        FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
        UNIQUE (task_id, attempt)
      )
    `)
    db.exec(`CREATE INDEX IF NOT EXISTS idx_task_runner_attempts_task ON task_runner_attempts(task_id, attempt DESC)`)
  }
}
```

Notes:
- All columns nullable except primary keys, task_id, attempt, started_at — runner posts exit info later via runner-exit (Plan 14-06); row is inserted at attempt START with `started_at` and updated at runner-exit with `exited_at`, `exit_code`, `failure_reason`, `stderr_tail`.
- UNIQUE (task_id, attempt) is the dedup key — claim route (Plan 14-05) INSERT ON CONFLICT DO NOTHING is safe on retry.
- FK CASCADE on task_id so task deletion cleans history; precedent: task_runner_tokens same pattern (migration 055).
- Do NOT touch existing tables; do NOT introduce non-null defaults that would break a backfill.
- Do not add to `migrations[]`; use `extraMigrations[]` per Phase 11/12 precedent.
