# Phase 18.1 Plan 02: Runner daemon + Admin config operator documentation — Summary

**One-liner:** Co-authored `docs/runtime/runner-daemon.md` (thin operator intro linking to the 154-line authoritative `scripts/README.runner.md`) and `docs/runtime/admin-config.md` (definitive reference for all eight `runtime.*` admin settings + secrets store + auth tiers + data-dir layout), with explicit blockquote-Warning callouts for the four drift-sensitive pitfalls (#4 project_repo_map EXCLUSIVE, #6 CONTAINER-01 secrets-off-argv, #7 MISSION_CONTROL_RECIPES_DIR for standalone, #8 recipe.secrets is ENV VAR NAMES not values).

## Objective Recap

Phase 18.1 is a pure-documentation phase closing v1.2 before `/gsd:complete-milestone v1.2` archives Phases 11–18. This plan co-authored two files because ~80% of the admin-config surface IS runner-config — splitting them across parallel plans risked contradictory statements (RESEARCH.md §Bundle B rationale). Target surfaces:

- **DOC-RUN:** Operator guide for `scripts/mc-runner.mjs` — boot sequence, env vars, exit codes, logs layout, troubleshooting. Kept thin; LINKS to `scripts/README.runner.md` for deep content.
- **DOC-CFG:** Definitive admin-configuration reference — all eight `runtime.*` keys, mount allowlist, project repo map, concurrency caps, secrets store, auth tiers, `.data/` layout, standalone-mode requirements, runner-config endpoint.

### Task 2 — `docs/runtime/admin-config.md` (commit `bb10ece`)

Authored the definitive admin-configuration reference. 269 lines. Structure follows Pattern 1:

- Header: source-of-truth citations (5 files), audience, prerequisites (admin-tier auth), section map (10 sections).
- "The 8 runtime.* settings": 4-column table enumerating all eight keys with per-key line-range citations to `src/app/api/settings/route.ts:60-99`, followed by a bash block showing the `pnpm mc settings set` pattern for each, followed by operator guidance prose for each key.

- "Secrets store": two consecutive blockquote Warnings — Pitfall #8 (recipe.secrets is ENV VAR NAMES with the `install -m 0600` pattern) and Pitfall #6 (CONTAINER-01 invariant restated with cross-link to runner-daemon.md).
