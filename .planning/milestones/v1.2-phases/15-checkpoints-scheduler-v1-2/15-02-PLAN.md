<objective>
Refactor the MC scheduler + task-dispatch + task-create API so recipe-tagged tasks flow through their own lanes:

1. **Scheduler (`src/lib/scheduler.ts`):** Cut `TICK_MS` from `60_000` to `30_000` (RESEARCH.md Focus Area 4 + Pitfall 3 — the locked 90s stale window = 3× 30s tick). Register a NEW `reconcile_runner_heartbeat` task with `intervalMs: 30_000` and wire it into the dispatch ladder (lines 442-460), the settings-key ladder (lines 425-436), the default-enabled ladder (line 437), `triggerTask` (line 513), and `getSchedulerStatus` (line 473).

2. **autoRouteInboxTasks (`src/lib/task-dispatch.ts:979-1076`):** Add a recipe-fast-path that pulls recipe-tagged inbox tasks FIRST, moves them `inbox → assigned` with `assigned_to` preserved (or NULL if not set), skips affinity scoring, emits `task.runner_requested` for each. Legacy scoring loop continues on the remainder.

3. **dispatchAssignedTasks (`src/lib/task-dispatch.ts:666-924`):** Add `AND t.recipe_slug IS NULL` to the SELECT at line 676 so recipe-tagged tasks are NEVER dispatched as legacy-agent jobs (RESEARCH.md Focus Area 5).

4. **requeueStaleTasks (`src/lib/task-dispatch.ts:588-664`):** Add a recipe-tagged branch that uses (a) `runner_heartbeats.last_heartbeat_at` freshness AND (b) `runner_heartbeats.metadata_json.active_task_ids` from the runner heartbeat metadata (populated in Plan 15-06). When a task's `recipe_slug IS NOT NULL`, do NOT use `agents.status` — use heartbeat + inventory instead.

5. **reconcileRunnerHeartbeat (`src/lib/task-dispatch.ts`, NEW export):** Pure function. Query `runner_heartbeats` for any row with `last_heartbeat_at >= now - 90`. If NO fresh heartbeat, find `in_progress` recipe-tagged tasks whose `updated_at < now - 90` and flip them to `assigned` atomically, emitting `task.runner_requested` per flip. Full implementation in RESEARCH.md § "Scheduler reconcile tick" code example.

6. **POST /api/tasks emission (`src/app/api/tasks/route.ts` after line ~468):** After the existing `eventBus.broadcast('task.created', parsedTask)`, when `parsedTask.status === 'assigned' && parsedTask.recipe_slug`, emit `eventBus.broadcast('task.runner_requested', { task_id, recipe_slug, workspace_id })` (RESEARCH.md Focus Area 5 "SCHED-05 direct-assigned emission point").

All three `task.runner_requested` emission points land in Wave 1:
- `autoRouteInboxTasks` (inbox → assigned) — THIS plan
- `POST /api/tasks` (direct-assigned with recipe_slug) — THIS plan
- runner-exit retry path (in_progress → assigned) — Plan 15-05 (Wave 2)
- Stale reconcile (in_progress → assigned) — THIS plan

Plan 15-01 MUST land first (same wave, but this plan's tests will fail to compile until event-bus.ts has the new union members). Treat the typecheck dependency as "land 15-01 Task 1 before running 15-02 tests" — both plans are in Wave 1 but 15-02 references types introduced in 15-01.

Purpose: The scheduler and dispatch lanes are the MC-side side of the runner control loop. Without this plan, no `task.runner_requested` SSE frames reach the daemon, which means no claim path exercises in Phase 15, which means no checkpoint traffic, which means Phase 15 can't be verified.

Output: Scheduler runs at 30s cadence with reconcile tick registered; dispatch lanes cleanly separated by recipe_slug; three emission points wiring `task.runner_requested`.

    ```ts
    export async function reconcileRunnerHeartbeat(): Promise<{ ok: boolean; message: string }> {
      const db = getDatabase()
      const nowUnix = Math.floor(Date.now() / 1000)
      const STALE_WINDOW_SECS = 90 // LOCKED: 15-CONTEXT.md § Heartbeat & Stale Detection (3× 30s tick)

      const fresh = db.prepare(
        'SELECT 1 FROM runner_heartbeats WHERE last_heartbeat_at >= ? LIMIT 1'
      ).get(nowUnix - STALE_WINDOW_SECS)

      if (fresh) {
        return { ok: true, message: 'Runner heartbeat fresh' }
      }

      // No runner fresh. Flip any in_progress recipe-task whose last update is
      // ALSO stale (prevents flipping a just-claimed task before the first heartbeat window elapses).
      const stuck = db.prepare(`
        SELECT id, recipe_slug, workspace_id FROM tasks
        WHERE status = 'in_progress' AND recipe_slug IS NOT NULL
          AND updated_at < ?
      `).all(nowUnix - STALE_WINDOW_SECS) as Array<{ id: number; recipe_slug: string; workspace_id: number }>

      if (stuck.length === 0) {
        return { ok: true, message: 'No stale in_progress recipe-tasks' }
      }

      let flipped = 0
      for (const t of stuck) {
        db.transaction(() => {
          const res = db.prepare(`
            UPDATE tasks
            SET status = 'assigned',
                container_id = NULL,
                runner_started_at = NULL,
                runner_last_failure_reason = 'runner_heartbeat_stale',
                updated_at = ?
            WHERE id = ? AND status = 'in_progress' AND recipe_slug IS NOT NULL
          `).run(nowUnix, t.id)
          if (res.changes > 0) flipped++
        })()

        if (flipped > 0) {
          eventBus.broadcast('task.runner_requested', {
            task_id: t.id,
            recipe_slug: t.recipe_slug,
            workspace_id: t.workspace_id,
          })
          eventBus.broadcast('task.status_changed', {
            id: t.id,
            status: 'assigned',
            previous_status: 'in_progress',
            reason: 'runner_heartbeat_stale',
          })
        }
      }

      return { ok: true, message: `Flipped ${flipped} stale recipe-task(s) back to assigned` }
    }
    ```
