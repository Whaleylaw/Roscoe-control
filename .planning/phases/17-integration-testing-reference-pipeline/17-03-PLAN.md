<objective>
Ship the D-02 direct-helpers integration test and the D-04 CI Docker wiring. This is the "fast lane" integration test — imports `runner-claim`, `runner-worktree`, `runner-docker` from `@/lib/*` directly, spawns the reference container via `spawnSync('docker', ...)`, and asserts DB + .mc/ + broadcast state at each hop. Companion plan 17-04 spawns the real daemon subprocess for the "full lane" fidelity. Both tests exercise the same pipeline; this plan is faster (target < 60s per run) and lives on the critical CI path.

Decision D-07: no `testcontainers` npm dep. Raw `child_process.spawnSync('docker', …)` per `scripts/mc-runner.mjs` precedent.

Decision D-04: quality-gate.yml gets a `pnpm mc:build-hello-world` step BEFORE `pnpm test` so the integration test finds the image. Also a `docker info` preflight so CI fails loud if Docker isn't available (vs the test silently skipping).

Purpose: Prove the full pipeline composes correctly through the production code paths EXCEPT for the daemon boot loop (covered in 17-04). Catches transport + claim + container-run + submit + Aegis integration bugs that in-process Phase 15 tests miss.

Output: One new Vitest integration test file (docker-gated, node env) + CI workflow additions.
