<objective>
Ship the RTEST-03 crash-recovery integration test. Kill a running container mid-task with `docker kill -s SIGKILL`, assert `.mc/` preservation, post runner-exit, re-claim, verify `is_resuming=true` and `prior_attempts.length===1`, re-seed via seedMcDir (which injects the LOCKED resume marker), re-run the container, assert BOTH attempt payloads are present in `.mc/progress.md` and `.mc/checkpoints.jsonl`.

Uses Pitfall 10 CMD override trick: `docker run --entrypoint /bin/sh mc-hello-world-agent:latest -c "sleep 30; node /app/agent.mjs"` to deterministically pause the container after the first checkpoint but before submit. Zero changes to the reference image.

Claude's Discretion per CONTEXT.md: crash-recovery ships as a SEPARATE plan (not merged with 17-04 daemon test) because the assertion surface is distinct and the single-test failure modes are easier to debug in isolation.

Purpose: Prove the crash-safety invariants ROADMAP SC-3 calls out — `.mc/` preservation + resume without redoing work. This is the correctness property that sells the entire runner design.
