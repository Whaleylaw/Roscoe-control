<objective>
Extend the runner-token submit route so task status flips `in_progress → review` (not directly to `done`). This is the D-01 LOCKED scope expansion that RTEST-02 asserts on. The existing runAegisReviews() scheduler task in src/lib/task-dispatch.ts already drives review → quality_review → done/revision/failed, so we only need to change the submit hop. No Aegis logic changes. No new endpoints. The route still:
  1. Requires runner-token principal (user.id === -2000)
  2. Cross-task 403 check unchanged
  3. Rate-limits via mutationLimiter
  4. Wraps status flip + revokeTokensForTask in a single db.transaction

Purpose: RTEST-02 and RTEST-04 both need the runtime to emit a `status='review'` task so downstream Aegis-approval assertions work. Without this change, the runtime goes straight to `done` and the ROADMAP SC-2 "task enters `review`, Aegis approves, task reaches `done`" flow is untestable.

Output: Production submit route modified + route-level unit test proving the review-flip and the existing invariants survive.
</objective>
