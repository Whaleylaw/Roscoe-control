**scripts/mc-runner-smoke.sh** - bash script that executes the end-to-end smoke.

Shebang + strict mode + help banner. Supports a primary sub-command `hello-world` (required for Phase 14). Future sub-commands documented as TODO: `preserve-on-stop`, `preserve-across-crash` (Phase 15/17).

Flow for `hello-world`:

1. Preflight: assert Docker running (`docker info > /dev/null` or exit 1 with message)
2. Preflight: assert MC reachable (`curl -s http://127.0.0.1:3000/api/health` returns 200 or exit 1)
3. Preflight: assert image exists locally (`docker image inspect mc-hello-world-agent:latest` or build it via `bash docker/hello-world-agent/build.sh`)
4. Resolve API_KEY: read `.data/api-key` or `.env` (existing helper pattern in `scripts/mc-cli.cjs` uses env MC_API_KEY) - use `MC_API_KEY=${MC_API_KEY:-$(cat .data/api-key 2>/dev/null || true)}`
5. Ensure the hello-world recipe is indexed: `curl -H "Authorization: Bearer $MC_API_KEY" http://127.0.0.1:3000/api/recipes/hello-world` returns 200; if 404, POST /api/recipes/resync and re-check
6. Configure `runtime.project_repo_map` to include the current repo path for a smoke project. If no smoke project exists, create one first via POST /api/projects. The smoke harness needs a project_id to put in workspace_source.
7. Ensure `runtime.mount_allowlist` includes the worktree parent dir (.data/runner/worktrees) - set via PUT /api/settings.
8. Create a task:
   ```bash
   TASK_ID=$(curl -s -X POST http://127.0.0.1:3000/api/tasks \
     -H "Authorization: Bearer $MC_API_KEY" -H "Content-Type: application/json" \
     -d '{"title":"smoke: hello world","recipe_slug":"hello-world","status":"assigned","workspace_source":{"project_id":<resolved>,"base_ref":"main"}}' \
     | jq -r '.id')
   ```
9. Start the runner in the background (`node scripts/mc-runner.mjs &`), capture PID.
10. Poll GET /api/tasks/$TASK_ID every 2s (up to 180s budget). Expected transitions: assigned -> in_progress -> done.
11. On timeout or status='failed' -> stop runner + exit 1 with diagnostic (tail the daemon.err log, show task.runner_last_failure_reason).
12. On success (status='done'):
   - Verify `.data/runner/worktrees/task-$TASK_ID/HELLO.md` exists (before GC destroys it - GC is 10min, so plenty of time)
   - Verify `.data/runner/logs/task-$TASK_ID/attempt-1/stdout.log` contains the hello-world agent's JSON log lines
   - Stop runner (kill PID)
   - Print a success banner
13. Cleanup: do NOT delete the task - leave it for inspection. Operator runs `docker ps -a` manually to confirm container was --rm'd.

Script structure:

```bash
#!/usr/bin/env bash
set -euo pipefail

case "${1:-help}" in
  hello-world) run_hello_world ;;
  help|--help|-h) usage; exit 0 ;;
  *) echo "Unknown subcommand: $1"; usage; exit 1 ;;
esac
```

Keep helpers as bash functions. Use `jq` for JSON parsing. If jq unavailable, document in README + fall back to `node -e 'process.stdout.write(JSON.parse(require("fs").readFileSync(0,"utf8")).id)'`.

Add a trap on EXIT that kills any lingering runner PID.
