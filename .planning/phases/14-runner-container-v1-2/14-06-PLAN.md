  <files>src/app/api/runner/tasks/[task_id]/runner-exit/route.ts</files>
  <action>
Implementation steps:

1. Auth: `requireRole('operator')` + `user.id === -1000` guard (403 otherwise).
2. Rate limit: `mutationLimiter(request)` per precedent.
3. Parse task_id from params; validate positive integer.
4. Zod body validation:
   ```typescript
   const Body = z.object({
     exit_code: z.number().int().nullable(),
     reason: z.enum(['exit', 'timeout', 'oom', 'crash', 'worktree_create_failed', 'docker_error']),
     stderr_tail: z.string().max(16_384).optional(),
     attempt: z.number().int().min(1),
   })
   ```
5. Load task: `db.prepare('SELECT * FROM tasks WHERE id = ?').get(taskId)`. 404 if missing.
6. Idempotency: if `task.status IN ('done','failed','cancelled')` return 409.
7. Resolve `runner_max_attempts` precedence: `task.runner_max_attempts ?? resolveRecipeMaxAttempts(task.recipe_slug) ?? 3` (LOCKED: `resolveRecipeMaxAttempts` is the filesystem re-parse helper exported from `@/lib/runner-claim`; do NOT hit getIndexedRecipeBySlug for this field).
8. Compute `failure_reason` string: `reason='exit' && exit_code !== null` -> `exit:${exit_code}`; `reason='timeout'` -> `timeout`; else just `reason`.
9. Classify:
   - `isSuccessfulExit = exit_code === 0 && reason === 'exit'`
   - `shouldFail = !isSuccessfulExit && ((task.runner_attempts >= resolvedMaxAttempts) || reason === 'worktree_create_failed')`
10. Transaction:
    ```typescript
    db.transaction(() => {
      const updRes = db.prepare(`
        UPDATE task_runner_attempts
        SET exited_at = ?, exit_code = ?, failure_reason = ?, stderr_tail = ?
        WHERE task_id = ? AND attempt = ?
      `).run(nowUnix, exitCode, failureReason, stderrTail ?? null, taskId, attempt)

      if (updRes.changes === 0) {
        // Defensive: attempt row was expected from claim route. Log and continue with status transition.
        // If this happens, the claim-route invariant is broken or the daemon posted the wrong attempt.
        logger.warn({ task_id: taskId, attempt }, 'runner-exit: task_runner_attempts UPDATE affected 0 rows; proceeding with status transition')
      }

      if (isSuccessfulExit) return

      if (shouldFail) {
        db.prepare(`
          UPDATE tasks SET status = 'failed', container_id = NULL, runner_last_failure_reason = ?, updated_at = ?
          WHERE id = ? AND status NOT IN ('done','failed','cancelled')
        `).run(failureReason, nowUnix, taskId)
        revokeTokensForTask(db, taskId, nowUnix)
      } else {
        db.prepare(`
          UPDATE tasks SET status = 'assigned', container_id = NULL, runner_started_at = NULL, runner_last_failure_reason = ?, updated_at = ?
          WHERE id = ? AND status = 'in_progress'
        `).run(failureReason, nowUnix, taskId)
      }
    })()
    ```
11. Return 204 on success.

Use `revokeTokensForTask` from `@/lib/runner-tokens`. Use `resolveRecipeMaxAttempts` from `@/lib/runner-claim`. Do NOT inline SQL revocation. Import `logger` from `@/lib/logger` for the defensive warn.
