   - Keep the existing `(? IS NULL OR project_id = ?)` and `(? IS NULL OR gsd_plan_id = ?)` patterns exactly as they are.
   - Append wave as a third optional filter.
   - Preserve the parameter binding order — each query takes `(workspaceId, ...identityParams, scope.projectId, scope.projectId, scope.gsdPlanId, scope.gsdPlanId, scope.wave, scope.wave)` with the wave binding twice (NULL-or-match idiom).
