Create 5 sub-components in `src/components/project/dashboard/`. All are `'use client'` components, use named exports, kebab-case filenames, `@/` imports per conventions. Each component receives data via props (not fetching internally).

**1. status-cards.tsx** (DASH-01, DASH-05 per D-03, D-12, D-13)
```
Props: { active: number; blocked: number; completed: number; onBlockedClick?: () => void }
```
- Export `StatusCards` function component
- Render 3 cards in a horizontal flex/grid row
- Each card: large count number + label below (using `t('dashboard.active')`, etc.)
- Blocked card: when `blocked > 0`, apply amber/warning styling — `bg-amber-500/10 border-amber-500/50` (per D-12). When `blocked === 0`, normal styling (per D-13).
- Blocked card: if `onBlockedClick` provided, card is clickable (cursor-pointer, onClick handler)
- Card styling: rounded border, padding, subtle background. Use Tailwind dark-mode-aware classes like `bg-zinc-800/50 border-zinc-700`

**2. progress-bar.tsx** (DASH-02 per D-05, D-06)
```
Props: { completed: number; total: number }
```
- Export `ProgressBar` function component
- Calculate percentage: `total === 0 ? 0 : Math.round((completed / total) * 100)`
- Render: label row with "Progress" text (i18n) + percentage, then a horizontal bar
- Bar: outer `bg-zinc-700 rounded-full h-2.5`, inner `bg-green-500 rounded-full h-2.5` with `style={{ width: \`${percentage}%\` }}`
- Below bar: text showing "{completed}/{total} tasks" using `t('dashboard.progressText', { completed, total })`
- When total is 0: show "No tasks yet" via `t('dashboard.noTasks')`

**3. health-badge.tsx** (DASH-06 per D-14, D-15)
```
Props: { blocked: number; total: number }
```
- Export `HealthBadge` function component
- Health logic (per D-15): `blocked === 0` -> On Track, `blocked / total < 0.25` -> At Risk, `blocked / total >= 0.25` -> Off Track. If `total === 0`, default to On Track.
- Render a text badge with emoji prefix per D-14:
  - On Track: green bg/text, prefix with a green circle or checkmark emoji
  - At Risk: amber bg/text
  - Off Track: red bg/text
- Badge styling: inline-flex, rounded-full, px-3 py-1, text-sm font-medium
- Use i18n keys: `t('dashboard.healthOnTrack')`, `t('dashboard.healthAtRisk')`, `t('dashboard.healthOffTrack')`

**4. project-brief.tsx** (DASH-03 per D-07, D-08)
```
Props: { description?: string | null; projectSlug: string }
```
- Export `ProjectBrief` function component
- Card with title "About" via `t('dashboard.about')`
- If description exists: render via `<MarkdownRenderer content={description} />` imported from `@/components/markdown-renderer`
- If no description (per D-08): show empty state with `t('dashboard.noDescription')` and a text link "Add one in Settings" that navigates to `/project/${projectSlug}/settings`
- Use `next/navigation` `Link` or `useRouter` for the settings link

**5. activity-feed.tsx** (DASH-04 per D-09, D-10, D-11)
```
Props: { activities: Activity[] }
```
- Export `ActivityFeed` function component
- Import Activity type from `@/store`
- Card with title "Recent Activity" via `t('dashboard.activityFeed')`
- Map over activities array (already limited to 20 by parent), render each as a row with:
  - Description text (from `activity.description`)
  - Relative timestamp using a local `relativeTime(created_at: number)` helper: seconds < 60 -> `t('dashboard.justNow')`, < 3600 -> `t('dashboard.minutesAgo', { count })`, < 86400 -> `t('dashboard.hoursAgo', { count })`, else `t('dashboard.daysAgo', { count })`
  - Type indicator: small text/emoji based on `activity.type` (e.g., "task_created" -> created, "task_updated" -> updated)
- If activities array is empty (per D-11): show `t('dashboard.noActivity')` empty state
- List items: separated by subtle borders, padding between entries

All components use `useTranslations('project')` for i18n.
