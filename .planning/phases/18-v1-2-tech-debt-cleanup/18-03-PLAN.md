<objective>
Correct the Phase 14 planning document drift flagged by v1.2 milestone audit item #3: plans and SUMMARYs describe the submit endpoint as "submit → done" but the actually-shipped behavior (per Phase 17-01 RTEST-02) is "submit → review", with Aegis approval then flipping review → done.

Purpose: Phase 14 plans were written before the Phase 17-01 design decision that moved the agent submit endpoint's post-condition from `status='done'` to `status='review'` (so an Aegis quality review can gate the final flip to done). The code now ships the review behavior, tests assert the review behavior, but Phase 14 narrative docs still say "done". The audit (audit-td-3) calls this out as correct behavior, stale docs. This plan corrects the prose in all seven affected Phase 14 markdown files — no code, no tests, no schema touched.

Output: Seven Phase 14 markdown files edited with narrative corrections. Each corrected location gets a brief inline note pointing to Phase 17-01 / RTEST-02 as the design authority so future readers understand why the review-flip exists.
