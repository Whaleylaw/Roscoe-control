<objective>
Write `docs/runtime/recipes.md` — the operator-facing authoring + indexing reference for the Mission Control recipe system.

Purpose: Today `scripts/README.runner.md` documents the runner daemon but there is NO operator doc for the recipe surface that sits upstream of it. Operators authoring a new recipe must currently reverse-engineer the Zod schema from `src/lib/recipe-schema.ts`, guess the indexer behavior from `src/lib/recipe-indexer.ts`, and discover the `POST /api/recipes/resync` endpoint by reading route files. This plan produces the canonical reference document.

Output: `docs/runtime/recipes.md` — a surface-scoped reference doc (Pattern 1 in 18.1-RESEARCH.md) with a top-of-file "Map" header, a Zod-schema-verbatim schema section, an indexer pipeline description, a recipe API endpoint table, a model-registry integration section, and explicit drift-guard callouts for pitfalls 3 (max_attempts filesystem-only) and 8 (secrets names-not-values).

Scope guardrail: DOCUMENTATION AUTHORING ONLY. No src/ code changes. No schema changes. No test changes. No migration changes.
</objective>
