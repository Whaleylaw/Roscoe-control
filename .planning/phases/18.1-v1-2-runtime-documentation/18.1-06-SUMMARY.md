## Decisions Made

- **Mermaid for the architecture diagram** rather than ASCII tree: text-reviewable, Git-diffable, renders natively on GitHub; the design spec already has an ASCII diagram so Mermaid differentiates the INDEX as "shipped behavior" rather than "design sketch."
- **Features block placed between Task Board and Memory Knowledge Graph** in README.md: recipes are task-adjacent (a recipe-tagged task becomes a runner container) so the semantic neighbor is correct; placing it after Framework Adapters (the plan suggestion) would have buried it too deep.
- **Design-spec banner inserted AFTER the H1 + author metadata**, not before: preserves attribution visual hierarchy (reader sees "this document exists, authored by X, dated Y" before the admonition) while still making the drift warning impossible to miss.
- **Plan's "Optional Edit 4" (recipes/ callout in architecture tree)** deliberately skipped: the existing tree shows concrete subdirectories inside `src/`, not top-level repo layout, and adding a `recipes/` entry at that indent would be misleading. Updated the adjacent "32 feature panels" comment instead — directly addresses the drift the plan flagged without forcing a style break.
- **cli-agent-control link-back uses a dedicated "See also" section** with 5 links, not a tight inline pointer: the CLI doc has NO `pnpm mc settings set runtime.*` examples today (grep returned zero hits), so the link-back section near the end is the correct surface.
