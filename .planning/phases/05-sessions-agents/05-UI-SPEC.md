# Phase 5 — UI Design Contract

> Visual and interaction contract for the Sessions and Agents scoped views inside the project workspace. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 5 adds two scoped views (`sessions-view.tsx`, `agents-view.tsx`) plus a nested session detail route. The Agents view is an embed-with-scope-prop over the existing `agent-squad-panel.tsx`. The Sessions view is a new two-section list component (chat threads then external runtime sessions). Session detail reuses `session-details-panel.tsx` with a `SessionDetailScope` prop. All visual tokens are inherited from the established Mission Control design system; this phase introduces no new colors, no new fonts, no new spacing exceptions.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (project-defined system via Tailwind + CSS variables) |
| Preset | not applicable |
| Component library | none — primitives are hand-rolled in `src/components/ui/` (button, loader, theme-background, etc.) on top of Tailwind |
| Icon library | **none** — CLAUDE.md forbids icon libraries. Use raw text or unicode emoji (e.g. `+`, `~`, `\u2713`, `!`, `\u2022`, `✅`, `⚠️`, `🔴`) as already done in `activity-feed.tsx` and `health-badge.tsx` |
| Font | `var(--font-sans)` (system UI stack, declared in `tailwind.config.js`); `var(--font-mono)` for ticket refs, timestamps, and runtime-type badges |

**Rationale for Tool: none** — CLAUDE.md project rules override the gsd-ui default. The codebase ships a mature design system using Tailwind plus HSL-based CSS variables (`--surface-0..3`, `--void-cyan|mint|amber|violet|crimson`, `--success`, `--warning`, `--info`, `--destructive`) defined in `src/app/globals.css` with dark-first values and light fallbacks. Four prior phases (1–4) have shipped views using these tokens. Introducing shadcn mid-project would fork conventions and violate the no-icon-library rule.

## Typography

Exactly 4 sizes and 2 weights — matching Tailwind classes already in use by `project-breadcrumb.tsx`, `status-cards.tsx`, `activity-feed.tsx`.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Meta | 12px | `text-xs` | 400 (regular) | 1.33 | Timestamps, runtime-type badges, "assigned" chip text, status-dot labels |
| Body | 14px | `text-sm` | 400 (regular) | 1.43 | Last-message preview, agent role, task count, empty-state body copy |
| Label | 14px | `text-sm` | 600 (semibold) | 1.43 | Agent name, thread row title (agent name for threads / ticket-ref for runtime sessions), section headers ("Chat threads", "External sessions") |

Weights allowed: **400 (regular)** and **600 (semibold)** only. No 500, 700, or italic. Monospace (`font-mono`) is used on meta-size text for ticket-refs and timestamps (12px regular), following `activity-feed.tsx:50`.

## Color

Mission Control ships a dark-first "void" palette. The 60/30/10 split is inherited from the existing design system and these phase 5 views must consume the same tokens — no new colors.

| Role | Token | Dark value (HSL) | Usage in this phase |
|------|-------|------------------|---------------------|
| Dominant (60%) | `bg-background` + `bg-surface-1` | `215 27% 4%` / `222 35% 7%` | Page background; empty-state card background; list row background |
| Secondary (30%) | `bg-card` / `bg-surface-2` + `border-border` | `220 30% 8%` / `220 30% 10%` + `220 20% 14%` | Agent cards, thread rows, runtime-session rows, section containers |
| Accent (10%) | `text-primary` / `bg-primary/10` (void-cyan `187 82% 53%`) | — | **Reserved-for list below** |
| Destructive | `text-destructive` / `bg-destructive/10` | `0 72% 51%` | Error-state text and icon-equivalent glyph; offline/failed status dot |

**Accent (void-cyan) is reserved for — and ONLY for:**

1. The primary CTA button fill in the Agents empty state (`bg-primary text-primary-foreground` on "Open Agents view")
2. The currently-selected session row's left border or background tint when a session detail is open (hover/selected state only)
3. The "assigned" chip on an explicitly-assigned agent card (`bg-primary/10 text-primary border-primary/30`) — this is the single accent use that visually distinguishes D-02's assignment badge

Accent is **not** used for:
- Non-selected row backgrounds
- Regular body text or headings
- Generic borders or dividers
- Timestamps, ticket-refs, or any metadata
- Runtime-type badges (those use neutral `bg-muted text-muted-foreground`)

**Semantic status-dot colors** (inherited from established palette, not counted in the 60/30/10 because they are status signals not brand accents):

| State | Token | Usage |
|-------|-------|-------|
| idle / running | `bg-success` (void-mint) | Agent status, session running |
| busy / at-risk | `bg-warning` (void-amber) | Agent busy, session warning |
| offline / failed | `bg-destructive` (void-crimson) | Agent offline, session failed |
| finished / neutral | `bg-muted-foreground` | Session finished cleanly |

Status dots are 8px circles (`w-2 h-2 rounded-full`), matching the pattern used in `agent-squad-panel.tsx` today.

## Interaction Contract (phase-specific additions to the visual spec)

### Row interactions

| Row kind | Click target | Destination | Focus ring |
|----------|--------------|-------------|------------|
| Chat thread row | Entire row (button semantics, `role="button"`, full-row `cursor-pointer`) | `/project/{slug}/sessions/{threadId}` via `router.push` with `startTransition` (matches `project-breadcrumb.tsx`) | `focus-visible:ring-2 focus-visible:ring-ring` |
| Runtime session row | Entire row | `/project/{slug}/sessions/{sessionId}` | Same |
| Agent card | Entire card (existing behavior inside embedded `agent-squad-panel.tsx`) | Agent detail flow (unchanged) | Inherited from global panel |

All row and card interactives must use `button` semantics or an actual `<button>` wrapper — matching the `StatusCards` pattern at `status-cards.tsx:30-31` which sets `role="button"` only when interactive.

### Hover states

- Chat-thread row: `hover:bg-surface-2` (subtle lift from `bg-surface-1`)
- Runtime-session row: same `hover:bg-surface-2`
- Transition: `transition-colors` only — **no scale, translate, or shadow transitions**. Matches the project-breadcrumb hover pattern.

### Selected/active states

When the session detail view is mounted for `sessionId = X`, and the sessions list is visible alongside (desktop), the matching list row gets `bg-primary/10 border-l-2 border-l-primary` (accent use case 2 above). This is the only selected-row treatment. On mobile/narrow widths the list is hidden when detail is open, so no selected state is needed.

### Real-time updates

When an SSE event causes a thread's `last_message_preview` or `last_message_at` to change, the row's last-message line must animate with the existing `animate-fade-in` keyframe (already declared in `tailwind.config.js`). No new keyframe introduced. No pulse or glow — that treatment is reserved for globally-critical indicators.

## Contract Inheritance From Prior Phases

This phase introduces no new design tokens. All values above are either (a) already present in `tailwind.config.js` / `globals.css` or (b) already used by Phase 1–4 components. The spec simply declares which tokens this phase commits to.

Concrete reuse points:

- **Empty state layout** — matches the shape used in Phase 3 (`activity-feed.tsx` empty branch at lines 35-42): heading + body + optional CTA, centered.
- **Row/card container** — matches `status-cards.tsx` (`rounded-lg border border-zinc-700 bg-zinc-800/50 p-4`) with the **one difference** that Phase 5 uses semantic tokens (`bg-card`, `border-border`, `bg-surface-2` on hover) instead of raw zinc scale. This is the correct forward-path — Phase 3's hard-coded zinc is legacy; new components use the tokens that support both light and dark modes.
- **Breadcrumb extension** — when a session detail is open, a fourth segment appears after the Sessions tab label. `project-breadcrumb.tsx` already supports arbitrary `view !== 'dashboard'`; extend the same branch to show `{ProjectName} > Sessions > {session label}` where the session label is either the agent name (for threads) or the ticket-ref (for runtime sessions).
- **Relative timestamps** — reuse `relativeTime()` and the existing `dashboard.justNow|minutesAgo|hoursAgo|daysAgo` keys from `activity-feed.tsx:10-22`. Do not fork a new implementation.
- **Status dots** — reuse the 8px dot pattern from `agent-squad-panel.tsx` with semantic tokens (`bg-success|warning|destructive`) rather than raw color names.
