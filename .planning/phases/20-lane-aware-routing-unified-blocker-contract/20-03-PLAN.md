```typescript
type TaskBlockerTransitionEvent = {
  task_id: number
  workspace_id: number
  direction: 'paused' | 'resumed'
  previous_status: string   // 'in_progress' on pause, 'awaiting_owner' on resume
  status: string            // 'awaiting_owner' on pause, 'assigned' on resume
  blocker_reason: string | null
  blocker_kind: 'needs_input' | 'needs_approval' | 'external_dependency' | 'policy' | 'other' | null
  resume_hint: string | null
  source: 'recipe' | 'legacy'
  attempt: number | null    // runner_attempts on recipe path; null on legacy
  ts: number                // unix seconds
}
```

Per-path field rules:
- Recipe pause: `blocker_reason` = checkpoint's blocker_reason trimmed; `blocker_kind` = null; `resume_hint` = null; `attempt` = task.runner_attempts; `source` = 'recipe'; `direction` = 'paused'.
- Recipe resume: `blocker_reason` = last-known reason from `runner_last_failure_reason` (best-effort — `blocked:<text>` string strip); `blocker_kind` = null; `resume_hint` = null; `attempt` = task.runner_attempts at resume time; `source` = 'recipe'; `direction` = 'resumed'.
- Legacy pause: all three envelope fields populated from the PUT body; `attempt` = null; `source` = 'legacy'; `direction` = 'paused'.
- Legacy resume: `blocker_reason`/`blocker_kind`/`resume_hint` parsed from the JSON envelope in `runner_last_failure_reason` BEFORE it is cleared (so the event carries the pre-clear values for observer context); `attempt` = null; `source` = 'legacy'; `direction` = 'resumed'.
