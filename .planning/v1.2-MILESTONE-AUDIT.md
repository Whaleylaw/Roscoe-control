## Cross-Phase Integration

Integration checker (gsd-integration-checker / sonnet) reports **47/47 connections wired, 0 orphans, 0 missing, 0 broken flows, 0 unprotected routes**. The post-Phase-18 re-check (2026-04-21) confirms Phase 18's changes — a one-attribute `data-testid` addition, a Playwright locator upgrade (`.or()` fallback), and doc-only corrections across seven markdown files — introduce zero cross-phase wiring disturbance. The authoritative integration proof (17-03/17-04 Vitest integration tests, 17-05 crash-recovery test, 17-06 Playwright spec) remains intact.

Five end-to-end flows traced with file:line evidence:
1. **Runtime pipeline:** recipe indexed → task created with recipe_slug → autoRouteInboxTasks → assigned → claim (atomic) → docker run with mounts+env → checkpoints (DB+JSONL) → submit → `status='review'` (atomic token revoke + broadcast) → Aegis → `done`
2. **Crash recovery:** container SIGKILL → runner-exit retry → re-claim `is_resuming=true` → LOCKED marker seeded → attempt-2 reads progress.md/checkpoints.jsonl
3. **UI reactivity:** `task.container_started/exited`, `task.checkpoint_added`, `recipe.indexed/removed` broadcast from API → SSE relay in `use-server-events.ts` → Phase 16 components (RecipeBadge, RunnerStatusBanner, ProgressTab, RecipesPanel) consume via DOM CustomEvents
4. **Blocker flow:** blocked checkpoint → atomic `awaiting_owner` flip + system comment → daemon `docker stop` on SSE → resolve → re-assigned → resume with LOCKED marker
5. **Auth boundaries:** runner principal scoped to `/api/runner/*`; runner-token verifies `:id` path match with 403 on cross-task; token revoked atomically on terminal transitions (terminal PATCH, submit review-flip, runner-exit terminal)
